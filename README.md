# Microsoft-365-Administration

This portfolio documents hands-on practice with core Microsoft 365 administration tasks using an administrative account in a non-production environment. The focus of this lab was to gain practical exposure to common Tier 1 / Tier 2 support activities, including user management, license assignment, MFA configuration, shared mailbox access, and sign-in log review.

All tasks were performed to mirror real-world support scenarios commonly encountered in enterprise and educational IT environments and are documented here for learning and demonstration purposes.


## Lab Objectives

- Practice real-world Microsoft 365 user administration
- Perform common Tier 1 / Tier 2 support tasks
- Gain familiarity with identity, licensing, security, and auditing features
- Document work in a clear, professional format suitable for a portfolio

---

## Tasks Performed

### User Account Management (Entra ID)
- Created a new user account in Microsoft Entra ID
- Verified account status and sign-in capability
- Unblocked a previously blocked user account

### License Management
- Assigned Microsoft 365 E5 licenses to users
- Verified service availability after license assignment

### Multi-Factor Authentication (MFA)
- Enabled multi-factor authentication for a user account
- Registered an authentication method
- Verified MFA enforcement during user sign-in

### Shared Mailbox Administration
- Created a shared mailbox for departmental use
- Granted user access permissions to the shared mailbox
- Verified mailbox availability in Outlook

### Security & Auditing
- Reviewed Microsoft Entra ID sign-in logs
- Identified successful and failed authentication attempts
- Used logs to validate MFA enforcement and account access
